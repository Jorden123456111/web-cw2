# 提交前最终检查清单（按扣分风险排序）

## A. 必过项（不满足会直接丢大分）
- [ ] 视频总时长不超过 5:00。
- [ ] 视频链接可访问（用无痕窗口打开验证）。
- [ ] 视频中完整演示 4 个命令：`build`, `load`, `print`, `find`。
- [ ] 展示多词查询和边界情况（空查询/不存在词）。
- [ ] GitHub 仓库公开可访问，包含源码、测试、README、requirements。
- [ ] Minerva 提交文档包含：视频链接 + GitHub 链接。
- [ ] 提交 index 文件（附件或下载链接）。

## B. 高分项（80+关键）
- [ ] 明确说明 6 秒 politeness window 在实现中如何保证。
- [ ] 解释倒排索引结构：`word -> doc -> {freq, positions}`。
- [ ] 展示测试运行结果（`pytest`）并说明覆盖了哪些边界。
- [ ] 展示提交历史体现增量开发，不是一次性提交。
- [ ] GenAI 反思有“具体例子 + 错误案例 + 你的修正 + 学习影响”。

## C. 展示质量项（容易被忽视）
- [ ] 终端字号足够大，代码和输出可读。
- [ ] 音频清晰、无明显噪音。
- [ ] 讲解顺序与评分结构一致（Demo -> Design -> Testing -> Git -> GenAI）。
- [ ] 每个小节控制时间，避免前面超时导致后面没讲完。
